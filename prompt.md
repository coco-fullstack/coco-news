## 役割

あなたは帳票テンプレート生成の専門家です。
提供されたPDFを解析し、データ差し込み用のWordテンプレート（.docx）を生成してください。

## 入力

- 帳票PDF（1ファイル）：{PDF_PATH}
- ※データはすべてダミー値として扱い、プレースホルダーに置き換えること

## 出力仕様

- 形式：.docx（python-docxで生成）
- 用紙サイズ：元PDFのレイアウトに応じて自動選択すること
  - 縦長帳票 → A4縦（幅11906 × 高さ16838 DXA）
  - 横長帳票 → A4横（幅16838 × 高さ11906 DXA）
  - 判定基準：列数が8列以上、または元PDFが横向きの場合はA4横を使用
- 余白：720 DXA（上下左右）
- フォント：**必ず「MS 明朝」を使用すること（MS GothicやArialは禁止）**
  - **重要：`run.font.name` だけでは日本語が文字化けする。以下のヘルパー関数を必ず定義し、全てのrunに適用すること：**

    ```python
    def _rfonts(run, font_name="MS 明朝"):
        """東アジアフォントを設定（文字化け防止）"""
        rPr = run._r.get_or_add_rPr()
        rf = rPr.find(qn('w:rFonts'))
        if rf is None:
            rf = OxmlElement('w:rFonts')
            rPr.insert(0, rf)
        for attr in ('w:eastAsia', 'w:ascii', 'w:hAnsi'):
            rf.set(qn(attr), font_name)
    ```

  - テキストを設定する際は、`run.font.name = "MS 明朝"` に加えて必ず `_rfonts(run)` を呼ぶこと
  - `from docx.oxml.ns import qn` と `from docx.oxml import OxmlElement` をインポートすること
- フォントサイズ：動的調整
  - 通常：本文18pt、タイトル32pt
  - 列数が10列超の場合：本文10pt〜12pt
  - 最小フォントサイズ：8pt（これ以下には縮小しない）
- セル幅の最小値：800 DXA
  - セル幅が800 DXA を下回る場合は、フォント縮小またはA4横への切り替えで対応すること
- 出力先：{OUTPUT_PATH}
- ※必ず指定されたパスに保存すること

## プレースホルダー仕様（最重要）

### 形式

{{snake_case_english}} 形式を使用すること。例：{{delivery_date}}

### 変換対象（必ずプレースホルダー化する項目）

- 日付・番号・コード類（発行日、伝票番号、バーコード等）
- 会社・担当者情報（名称、住所、電話番号等）
- 業務固有データ（品名、数量、金額等）
- ページ情報（ページ番号、総ページ数）

### 固定値として残す項目（プレースホルダーにしない）

- 項目ラベル（「業者名」「納入日」等のヘッダーセル文字）
- 帳票タイトル（「運搬指示書」等）
- 単位記号・記号類（「×」「〒」「Tel：」等の固定文字）
- 備考欄の定型文言

### 明細繰り返し行の命名規則

行番号をサフィックスに付与する。
例：detail_1_product_name, detail_1_quantity, detail_2_product_name ...
最大行数はPDFから読み取った行数に合わせること。

## レイアウト再現ルール

- テーブル構造・セル結合はPDFに忠実に再現する
- ヘッダーセルは灰色背景（fill: "D9D9D9"）、中央揃え・太字
- 値セルは白背景、左揃え（数値列は右揃え）
- セル内余白：top/bottom 60、left 120、right 100（DXA）

## 成果物として必ず出力するもの

1. .docxファイル（{OUTPUT_PATH} に保存）
2. 処理完了後、保存パスを明示すること

## 禁止事項

- \n による改行（Paragraphを分けること）
- WidthType.PERCENTAGE の使用（DXA固定）
- unicode bullet文字の直接埋め込み
- セル幅不足による文字の1文字ずつ縦分解（必ずセル幅800 DXA以上を確保すること）
- Pythonスクリプト（.pyファイル）の生成（直接 {OUTPUT_PATH} に .docx を保存すること）
- 中間ファイルや補助スクリプトの作成（1ステップで直接docxを生成・保存すること）
